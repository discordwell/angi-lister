Netic Angi Lead API

Tenant and Admin API for managing leads, personalization, and platform configuration.

Contents Authentication Error Format Tenant API Admin API Personalization Config

Authentication

All API requests require a Bearer token in the Authorization header. API keys are generated by a platform admin.

Authorization: Bearer angi_a1b2c3d4_e5f6789...

There are two types of API keys:

  • Tenant keys — scoped to a single tenant. Can access the /api/v1/tenant/* endpoints.
  • Admin keys — platform-wide access. Can access the /api/v1/admin/* endpoints.

Example: Get your tenant profile

curl -H "Authorization: Bearer angi_a1b2c3d4_..." \
     {{ base_url }}/api/v1/tenant/me
API keys are returned once at creation. Store them securely — they cannot be retrieved later.

Error Format

Errors return standard HTTP status codes with a JSON body:

{
  "detail": "Lead not found"
}
  • 401 — Missing or invalid API key
  • 403 — Insufficient permissions (e.g., tenant key on admin endpoint)
  • 404 — Resource not found (or not owned by your tenant)
  • 409 — Conflict (e.g., duplicate slug or mapping)
  • 422 — Validation error

Tenant API

All endpoints under /api/v1/tenant. Requires a tenant API key.

Profile

GET/api/v1/tenant/me
Returns your tenant profile (name, slug, email, phone, brand color, personalization status).

Leads

GET/api/v1/tenant/leads?limit=50
List your recent leads. Returns lead ID, name, email, category, urgency, status, timestamp.
GET/api/v1/tenant/leads/{lead_id}
Full lead detail including events, outbound messages, and duplicate matches.

Metrics

GET/api/v1/tenant/metrics
KPI dashboard: leads (24h / total), median speed-to-lead, delivery rate, duplicate rate, conversion rate.

Duplicates

GET/api/v1/tenant/duplicates?limit=100
List duplicate lead pairs with similarity scores and evidence.
GET/api/v1/tenant/duplicates/export
Download CSV of duplicate matches for rebate claims.

Personalization Config

GET/api/v1/tenant/config
Full personalization configuration: sample email, pricing tiers, home bases, job rules, specials.
PUT/api/v1/tenant/config
Update config fields (partial update — only send fields you want to change).

Example: Update config

curl -X PUT -H "Authorization: Bearer angi_..." \
     -H "Content-Type: application/json" \
     -d '{"personalization_enabled": true, "sample_email": "Thanks for reaching out!"}' \
     {{ base_url }}/api/v1/tenant/config

Home Bases

POST/api/v1/tenant/home-bases
Add a service location. Body: {"name": "Main Shop", "lat": 38.6, "lng": -90.2, "address": "..."}
DELETE/api/v1/tenant/home-bases/{id}
Remove a home base.

Job Rules

POST/api/v1/tenant/job-rules
Add a rule. Body: {"category_pattern": "HVAC", "rule_type": "whitelist"}
Rule types: whitelist, blacklist, wantlist
DELETE/api/v1/tenant/job-rules/{id}
Remove a job rule.

Specials

POST/api/v1/tenant/specials
Add a promotion. Body: {"name": "Spring Tune-Up", "discount_text": "$49", "conditions": {"category_contains": "AC"}}
PUT/api/v1/tenant/specials/{id}
Update a special (partial update).
DELETE/api/v1/tenant/specials/{id}
Remove a special.

Admin API

All endpoints under /api/v1/admin. Requires an admin API key or operator console session.

Tenants

GET/api/v1/admin/tenants
List all tenants on the platform.
POST/api/v1/admin/tenants
Create a new tenant. Body: {"name": "Acme HVAC", "slug": "acme-hvac", "email": "..."}
PUT/api/v1/admin/tenants/{id}
Update a tenant's profile fields.

API Keys

GET/api/v1/admin/tenants/{id}/api-keys
List active API keys for a tenant.
POST/api/v1/admin/tenants/{id}/api-keys
Generate a new API key. Body: {"name": "CLI key"}. Returns the raw key once.
DELETE/api/v1/admin/tenants/{id}/api-keys/{key_id}
Revoke an API key (soft delete).

Example: Create a tenant key

curl -X POST -H "Authorization: Bearer angi_admin_..." \
     -H "Content-Type: application/json" \
     -d '{"name": "CLI integration"}' \
     {{ base_url }}/api/v1/admin/tenants/TENANT_ID/api-keys

# Response:
{
  "id": "...",
  "name": "CLI integration",
  "key_prefix": "angi_a1b2c3d4",
  "raw_key": "angi_a1b2c3d4_e5f6789..."  // Save this! Only shown once.
}

Mappings

POST/api/v1/admin/tenants/{id}/mappings
Map an Angi ALAccountId to a tenant. Body: {"al_account_id": "123456"}

Global Data

GET/api/v1/admin/metrics
Platform-wide KPI metrics across all tenants.
GET/api/v1/admin/leads?limit=50
List all leads across all tenants.

Personalization Config

When personalization_enabled is true, incoming leads are processed through a 3-pass AI pipeline before outreach:

  1. Repeat customer check — Queries for same email/phone in last 7 days. If found, the LLM decides whether to send or skip.
  2. Job desirability — Checks the lead's category against your job rules:
    • whitelist — Only respond to these categories (if any whitelist rules exist)
    • blacklist — Never respond to these categories
    • wantlist — Respond with extra enthusiasm
  3. Offer selection — Geocodes the lead's address, computes distance to your home bases, applies pricing tiers and qualifying specials.

Pricing Tiers

Set via PUT /api/v1/tenant/config in the pricing_tiers field:

[
  {"max_mi": 1, "text": "$39 diagnostic"},
  {"max_mi": 5, "text": "$59 diagnostic"},
  {"max_mi": 15, "text": "$79 diagnostic"}
]

Special Conditions

The conditions field on specials supports:

{
  "category_contains": "Water Heater",   // lead category must contain this
  "max_distance_mi": 10.0,               // lead must be within this distance
  "urgency_in": ["Today/Emergency"],     // lead urgency must match
  "valid_after": "2026-01-01",           // date range
  "valid_before": "2026-12-31"
}
